Creating a New Agent

1. Create via the Web UI

Click + New in the agents sidebar. Fill in name, instructions, model, and cadence. Submitting runs scaffold.createAgent() which creates all the files below automatically.

You can also create via Slack with $agents create — same result.

2. What gets created automatically
~/.claude/agents/<Name>.md — agent file
---
fileClass: Agent
agent-name: Name
status: Active
model: claude-haiku-4-5-20251001
cadence: On demand
slack-channel:          ← fill in manually
last-session:
tags:
  - agents
---
claude-workspaces/<name>/CLAUDE.md — Claude's entry point
Before doing anything, read `~/.claude/agents/<Name>.md` for your full instructions.
claude-workspaces/<name>/.claude/settings.json — tool permissions
Default: Read, Edit, Write, Glob, Grep
3. Fill out the agent file (~/.claude/agents/<Name>.md)

The scaffold creates a minimal shell. Key sections:

Identity Who the agent is, its purpose, its voice.
Instructions What it reads, what it does, how it communicates. Use PostMessage to send to Slack, WriteCard to log to vault, SpawnAgent to delegate.
Knowledge Domain context baked in.
4. Set a trigger (optional)

Go to the Jobs tab and create a new job. Select the agent and action, set a cron schedule, and save.

5. Add action templates (optional)

Create ~/.agents/actions/<Action>.md with agents: metadata for shared global actions, or use <workspace>/.agents/actions/ for workspace actions. These appear in the agent's Overview tab and in the Jobs picker.

6. Add a shell script (optional — background agents)

If the agent needs file discovery or computed inputs before running, create admin/Meta/run-<name>.sh. It will appear in the agent's Overview tab. The scheduler job should call the shell script directly.

Quick reference
File Purpose Who writes it
~/.claude/agents/<Name>.mdIdentity, instructions, knowledgeYou (scaffold creates shell)
workspaces/<name>/CLAUDE.mdClaude's entry pointScaffold
workspaces/<name>/.claude/settings.jsonTool permissionsScaffold
~/.agents/actions/<Action>.mdShared global action templates with optional agents: metadataYou (optional)
admin/Meta/run-<name>.shPre-run file discovery / input prepYou (optional, background only)
